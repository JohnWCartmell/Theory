
\section{Dust Jacket}

Entity modelling is widely understood in software development but too rarely used effectively because there has traditionally been a discontinuity between modelling and data definition --- a gap that, at best, has been bridged by a corrective step presented as data normalisation. This book shows that the discontinuity need never have existed. By taking account of equivalent paths within the system of relationships that characterises any subject domain, the transition from entity modelling to data definition becomes seamless. What has long been accepted as normal practice can be consigned to the past.

This book presents entity modelling as it should always have been: clear, rigorous, and direct.
